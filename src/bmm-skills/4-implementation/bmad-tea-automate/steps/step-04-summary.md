## STEP GOAL

Run the generated tests, fix failures, and produce a final automation summary.

## RULES

- Tests MUST be run — do not skip validation
- If a test fails, investigate the root cause: is it a test bug or a real code issue?
- Test bugs: fix the test (max 3 fix attempts per test, then mark as `test.fixme()` / `@pytest.mark.skip` / equivalent with a detailed comment)
- Real code issues: report them in the summary — do not modify production code
- The automation summary must be written to a file in the project

## SEQUENCE

### 1. Run tests

Execute the generated tests using the project's test runner:

**Common commands:**
- Vitest: `npx vitest run --reporter=verbose`
- Jest: `npx jest --verbose`
- Playwright: `npx playwright test --reporter=list`
- pytest: `python -m pytest -v`
- Go: `go test -v ./...`

Run ALL generated test files. Capture:
- Total tests run
- Passing count
- Failing count
- Error messages and stack traces for failures

### 2. Heal failing tests

For each failing test:

1. **Read the error** — understand the failure reason
2. **Classify the failure:**
   - **Selector issue** — element not found, wrong selector
   - **Timing issue** — race condition, element not ready
   - **Data issue** — unexpected data format, missing seed data
   - **Network issue** — API call failed, mock not set up
   - **Logic issue** — test assertion wrong, implementation different from expected
   - **Real bug** — the production code is actually broken
3. **Fix test bugs** (categories 1-5):
   - Apply the fix
   - Re-run the specific test
   - Max 3 attempts per test
4. **Report real bugs** (category 6):
   - Do NOT modify production code
   - Document in the summary

If a test cannot be fixed after 3 attempts, mark it:
- JS/TS: `test.fixme('Reason: ...')`
- Python: `@pytest.mark.skip(reason='...')`
- Go: `t.Skip("...")`

Include a detailed comment explaining:
- What failure occurred
- What fixes were attempted
- Why they didn't work
- Manual investigation steps needed
